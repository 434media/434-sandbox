# Sales Deck — brief

A **web-based sales deck** — the 434 Media / Digital Canvas pitch as a scrollable or slide-style web page, not a PDF. Build it at the [`/sales-deck`](../app/sales-deck) route.

## What you're building

A polished, on-brand deck that tells the story and can be opened with a link. The narrative centers on the **health vertical**, with **Lab Cafe as the customer example** (the real 434 customer placed in the deck); the rest of the assets are about 434 / Digital Canvas.

Think about:
- A strong hook and a clear narrative arc (problem → approach → proof → call to action)
- Lab Cafe as the worked example of how 434 develops an opportunity in health
- On-brand type, color, and imagery (coordinate with the Storytellers squad on assets)
- Works well full-screen and on a phone; easy to navigate

## Good to know

- **No production coupling.** This is a standalone marketing page — it can be **deployed on its own** straight from this repo. Nice early win: get a deployable skeleton up, then iterate.
- Keep content easy to edit (data/markdown-driven if you can) so copy changes don't mean code surgery.

## Done looks like

A reviewed, deployable deck (feature branch → PR → merged to `main`) that someone could send to a prospect. Drop the preview link in the PR and in Discord.
